# Redmine MCP Documentation

Welcome to the Redmine MCP documentation. This directory contains guides and examples for using the Redmine MCP integration.

## Documentation Overview

- [Basic Usage Guide](basic-usage.md): Comprehensive guide to getting started with Redmine MCP
- [Quick Reference](quick-reference.md): Quick reference for common operations
- [Examples](examples/): Directory containing example scripts demonstrating MCP usage

## Getting Started

If you're new to Redmine MCP, start with the [Basic Usage Guide](basic-usage.md). This guide will walk you through the basics of setting up and using the Redmine MCP integration.

## Examples

The [examples](examples/) directory contains practical scripts that demonstrate how to use the Redmine MCP integration:

- [Basic Operations](examples/basic-operations.js): Examples of common operations like listing projects, creating issues, etc.
- [Common Workflows](examples/common-workflows.js): Examples of workflows like adding time tracking, changing issue status, etc.

## Quick Reference

The [Quick Reference](quick-reference.md) provides a concise summary of the most common operations, including their parameters and return values. It's designed to be a quick lookup resource for experienced users.

## Project Structure

The Redmine MCP project is organized into several subprojects:

1. **MCP Project**: The main project (ID: 1)
2. **Hosted**: The hosted environment configuration (ID: 4)
3. **bugs**: Bug tracking and issue management (ID: 5)
4. **docs**: Documentation for the MCP project (ID: 6)
5. **features**: New feature development (ID: 7)

Each subproject contains issues related to its specific area of focus.

## Help and Support

If you encounter any issues or have questions about using the Redmine MCP integration, please create a new issue in the [bugs](http://localhost:3000/projects/bugs/issues) subproject.

## Contributing

Contributions to the documentation are welcome. If you find an error or would like to suggest an improvement, please create a new issue in the [docs](http://localhost:3000/projects/docs/issues) subproject.
